# 지뭐듣?

- **커밋 규칙**
  - 예시) Feat: message
  - ```Feat``` : 새로운 기능 추가
  - ```Fix``` : 버그 해결
  - ```Docs``` : README, 주석 작성
  - ```Style``` : 스타일 관련 기능(코드 포맷팅, 세미콜론 누락, 코드 자체의 변경이 없는 경우)
  - ```Refactor``` : 코드 개선, 리팩토링
  - ```Test``` : 테스트 코드 추가 및 리팩토링
  - ```Chore``` : 빌드 업무 수정, 패키지 매니저 수정 등 (ex. gitignore 수정)
